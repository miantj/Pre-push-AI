# AI Code Review 使用指南

## 一、安装与启用

1. 安装 VSIX → 重载 Cursor
2. 打开项目根目录（含 `.git`）
3. `Cmd+Shift+P` → **AI Code Review: 启用审查**

完成后 `.cursor/` 下会有：

- **`ai-code-review.说明.md`** ← 推荐先看，用人话解释配置
- **`ai-code-review.json`** ← 实际配置，编辑时有中文提示

---

## 二、日常使用

| 场景 | 怎么做 |
| --- | --- |
| push 前自动审 | 默认已装好 `pre-push` hook，直接 `git push` |
| 提交前先审 | 设置 `hooks` 加 `pre-commit`，或手动 **运行审查** |
| 不 push 先审 | `Cmd+Shift+P` → **运行审查** → 选范围 |
| 看审查结果 | **查看上次报告**，或打开 `.cursor/ai-code-review-last.md` |

**运行审查** 会问你审哪段代码：

- **相对目标分支** — 本分支相对 main/stable 改了什么
- **未提交变更** — 还没 commit 的改动

## 三、配置怎么改？

### 推荐：直接改 json

打开 `.cursor/ai-code-review.json`，鼠标悬停字段可看中文说明。

**禁用审查**：把 `"enabled": false` 即可。

### 看不懂 json？

打开同目录的 **`ai-code-review.说明.md`**，里面有当前配置解读、字段对照表和 Provider 配置教程。

### 进阶：Cursor 设置（可选）

仅在没有 `ai-code-review.json`、第一次启用前，可用设置面板预览默认值。启用后以 json 为准。

## 四、配置项人话对照

| 设置 | 人话 | 一般怎么设 |
| --- | --- | --- |
| enabled | 总开关 | true |
| hooks | 何时自动审 | `["pre-push"]`；不要自动审就 `[]` |
| reviewMode | Agent 还是 API | 默认 agent |
| agent | Cursor 或 Claude | 默认 cursor |
| provider | 正在用的那一个 API 模型 | 见下文 Provider 教程 |
| baseline | 分支跟谁比 | 保持 auto |
| defaultScope | 手动审默认范围 | branch |

## 五、Agent vs Provider

**Agent（默认）**  
本机 `agent` 或 `claude` 命令，跟 Cursor 订阅走。

**Provider**  
用自己的 API Key，适合 Cursor 额度不够时。**一次只配一个模型**，写在 json 的 `provider` 里即可。

### Provider 最小配置

```json
{
  "reviewMode": "provider",
  "provider": {
    "type": "deepseek",
    "model": "deepseek-chat"
  }
}
```

`baseUrl`、`path` 留空时会按 `type` 自动用平台默认值。

### 换平台 / 换模型

只改 `provider` 这一段：

| 平台 | type | model 示例 | baseUrl（可选） | path（可选） |
| --- | --- | --- | --- | --- |
| DeepSeek | deepseek | deepseek-chat | https://api.deepseek.com | /v1/chat/completions |
| MiniMax 国际 | minimax | MiniMax-Text-01 | https://api.minimax.io | /v1/text/chatcompletion_v2 |
| MiniMax 国内 | minimax | MiniMax-Text-01 | https://api.minimaxi.com | /v1/text/chatcompletion_v2 |
| OpenAI | codex | gpt-4o | https://api.openai.com | /v1/chat/completions |

### MiniMax 国内完整示例

```json
{
  "enabled": true,
  "reviewMode": "provider",
  "provider": {
    "type": "minimax",
    "model": "MiniMax-Text-01",
    "baseUrl": "https://api.minimaxi.com",
    "path": "/v1/text/chatcompletion_v2"
  }
}
```

### API Key 怎么配？

1. **扩展内手动审查**：`Cmd+Shift+P` → **设置 Provider API Key**（存 SecretStorage）
2. **Git hook 自动审查**：在 `.cursor/ai-code-review.env` 写 `AI_CODE_REVIEW_API_KEY=你的key`（hook 读不到 SecretStorage）

## 六、常见问题

**push 没跑审查？** → 是否 **启用审查**，`enabled` 是否为 true

**FAIL 但要紧急推送？** → `AI_CODE_REVIEW_ALLOW_ISSUES=1 git push`

**Agent 额度用尽？** → 换 Claude 或 Provider 模式

**Provider 请求失败？** → 检查 `provider.type` / `model` / `baseUrl` / `path`，以及 API Key 是否有效
