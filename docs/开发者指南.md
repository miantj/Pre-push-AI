# AI Code Review 开发者指南

面向维护者：本地开发、调试、打包与发布。

---

## 1. 环境要求

- Node.js 18+（推荐 20+）
- npm
- Git
- 打包 VSIX 需要 `@vscode/vsce`（脚本会自动用 `npx` 兜底）

Agent 模式本地调试还需：

- Cursor Agent CLI：`agent --version`
- 或 Claude Code：`claude --version`

---

## 2. 项目结构

```
cursor-pre-push-review/     # 审查 CLI（TypeScript → dist/）
cursor-pre-push-vscode/     # VS Code 扩展（TypeScript → out/）
scripts/package-vscode.sh   # 一键 VSIX
docs/                       # 文档
```

CLI **不发布 npm**，通过扩展的 `bundle:review` 脚本复制进 `node_modules/ai-code-review/` 再打入 VSIX。

---

## 3. 本地开发

### 3.1 构建 CLI

```bash
cd cursor-pre-push-review
npm install
npm run build
```

直接运行：

```bash
node dist/cli.js review --scope uncommitted
# 或
./bin/ai-code-review review --scope branch
```

CLI 会自动加载当前目录下 `.cursor/ai-code-review/env`（dotenv）。

### 3.2 开发扩展

```bash
cd cursor-pre-push-vscode
npm install
npm run compile          # 仅编译扩展
npm run bundle:review    # 构建 CLI 并复制到 node_modules/ai-code-review
```

在 Cursor 中：

1. 打开 `cursor-pre-push-vscode` 文件夹
2. `F5` 启动 Extension Development Host
3. 在宿主窗口打开要测试的 Git 仓库
4. 执行「启用审查」等命令

### 3.3 调试 Git Hook

启用审查后检查：

```bash
cat .husky/pre-push    # 或 .git/hooks/pre-push
cat .cursor/ai-code-review/hook.sh
```

手动模拟 hook：

```bash
.cursor/ai-code-review/hook.sh run --scope branch
echo $?   # 0=通过, 1=拦截
```

带环境变量调试：

```bash
AI_CODE_REVIEW_FROM_HOOK=1 node cursor-pre-push-review/dist/cli.js run --scope branch
```

## 4. 打包 VSIX

```bash
./scripts/package-vscode.sh
```

步骤：

1. `cursor-pre-push-review` → `npm run build`
2. `cursor-pre-push-vscode` → `npm run compile` + `npm run bundle:review`
3. `vsce package` → 生成 `cursor-pre-push-vscode/*.vsix`

安装测试：

```text
Cursor → 扩展 → ··· → 从 VSIX 安装
```

## 5. 关键扩展点

| 想改什么 | 改哪里 |
| --- | --- |
| 默认审查 prompt | `cursor-pre-push-review/src/prompt.ts` → `DEFAULT_REVIEW_PROMPT` |
| 默认 config 字段 | `cursor-pre-push-review/src/config.ts` 与 `cursor-pre-push-vscode/src/settings/settingsProvider.ts` 的 `DEFAULTS` |
| Provider 平台预设 | `cursor-pre-push-review/src/provider.ts`、`cursor-pre-push-vscode/src/shared/providerPresets.ts` |
| config.json Schema 提示 | `cursor-pre-push-vscode/schemas/ai-code-review.schema.json` |
| 用户可读配置说明 | `cursor-pre-push-vscode/src/shared/configLabels.ts`（生成 `ai-code-review.说明.md`） |
| Hook 模板 | `cursor-pre-push-vscode/src/infrastructure/hookInstaller.ts` → `buildHookContent` |
| 跳过审查的路径规则 | `cursor-pre-push-review/src/git.ts` → `REVIEW_SKIP_PATH_RE` |
| 报告格式 | `cursor-pre-push-review/src/report.ts` |
| Verdict / 基础设施错误识别 | `cursor-pre-push-review/src/verdict.ts` |

修改 prompt 默认值后，已存在的工作区 `prompt.md` **不会**自动覆盖；用户需执行「恢复默认 Review Prompt」。

## 7. 测试建议

当前仓库无自动化测试套件，建议手动回归：

| 场景 | 预期 |
| --- | --- |
| 无变更 push | 跳过审查，exit 0 |
| 小改动 push | 单批审查，写报告 |
| 大 diff | 控制台显示分批，汇总 verdict |
| `enabled: false` | 不跑审查 |
| JSON 语法错误 | 扩展移除 hook，状态栏提示 |
| Provider 无 Key | 扩展警告并关闭 hook |
| `review` 命令 FAIL | exit 1 但不拦 git（非 hook） |

## 8. 常见问题（开发）

**扩展找不到 CLI**  
→ 先 `npm run bundle:review`，确认 `node_modules/ai-code-review/dist/cli.js` 存在。

**hook 跑了但扩展没反应**  
→ hook 走独立 shell 进程，看终端输出和 `.cursor/ai-code-review-last.md`。

**修改 CLI 后扩展未更新**  
→ 重新 `bundle:review` 并重启 Extension Host。

**vsce package 失败**  
→ 检查 `publisher` 字段；脚本使用 `--allow-missing-repository`。
